Handoff-промпты для следующих slice'ов admin-v2 development.

Исторические `SLICE-*` файлы в этой папке в основном оформлены как self-contained ultra-strict prompts для нового чата (Agent mode). Для новых handoff'ов теперь используйте compact authoring kit ниже.

Дальнейшие slice'ы — post-MVP-3 expansion (расширение allowlist'ов, occurrence/sharing модели, additional threshold types/delivery channels) либо MVP-4 prep (cases, multi-step workflows). ## Conventions

- Файлы названы по схеме `SLICE-<phase>-<slug>.md`
- Исторические self-contained prompts в этой папке часто включают полный inline-набор секций: mission, hard rules, sources of truth, capability/role mapping, schema/index plan, API contracts, frontend contracts, test strategy, gate updates, full file list, definition of done, anti-scope, escalation triggers, starter command, decision rationale evidence
- Для новых handoff'ов этот полный inline-набор больше не обязателен: используйте compact structure из `HANDOFF-BASE.md` и наследование через `HANDOFF-SHARED-MODULES.md`
- Prompts not normative for the codebase; нормативные документы — это `admin-v2-pack/*.md` и `docs/admin-v2-merge-gates.md`. Handoff просто выстраивает one-shot execution plan для конкретного slice'а

## Authoring Guidance

Для новых handoff'ов не копируйте старый fully self-contained стиль целиком. Собирайте новый prompt через authoring kit:

- `HANDOFF-BUILD-RULES.md` — правила, что держать inline, что выносить в shared docs, и как ужимать token-heavy sections без потери safety rails
- `HANDOFF-BASE.md` — конкретный compact template для нового `SLICE-*` handoff
- `HANDOFF-SHARED-MODULES.md` — named common/family modules, которые можно наследовать вместо повторения governance-layer и семейной rationale

Ожидаемый режим authoring'а: `HANDOFF-BASE.md + slice delta + inherited modules`.

Сохраняйте inline только то, что реально предотвращает ошибочную реализацию:

- mission и точную границу slice
- slice-specific non-negotiables
- exact SQL / CHECK / allowlist / query predicates, если literal shape contract-critical
- новые security-sensitive или money-sensitive decisions
- exact verification commands / pass conditions / reconciliation tokens

Повторяющиеся hard rules, gate boilerplate, anti-scope lists, reconciliation-note scaffolding и family rationale переносите в shared docs и ссылайтесь на них по имени модуля.

## Writer Helpers

Дополнительные writer-helper файлы не меняют authoring rules, а только помогают скормить их агенту:

- `HANDOFF-WRITER-PROMPT.md` — manual mode: вы сами даёте агенту основной prompt и отдельно подставляете context
- `HANDOFF-WRITER-CONTEXT-TEMPLATE.md` — шаблон для подготовки context к `HANDOFF-WRITER-PROMPT.md`
- `HANDOFF-WRITER-AUTONOMOUS-PROMPT.md` — single-shot mode: агент сам читает kit, сам нормализует raw brief и сам пишет handoff

Если нужен один файл без ручной сборки prompt + context, используйте `HANDOFF-WRITER-AUTONOMOUS-PROMPT.md`.
